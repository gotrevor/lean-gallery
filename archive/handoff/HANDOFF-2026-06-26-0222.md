## 🧠 Context to carry forward
The spec's **primary plan was infeasible as written** and I pivoted — the successor must
understand why, since a naive re-read of the spec would send them down the dead end:

- **`Ordinal.nadd` (natural/Hessenberg sum) is GONE from mathlib.** `SetTheory/Ordinal/
  NaturalOps.lean` (702 lines) was deleted in mathlib #35550; absent at the v4.31.0 pin
  (`grep -r nadd` in `.lake/packages/mathlib` → nothing; `♯` now means inner-product
  sharp). The spec's `o(node) = ♯ ωᵒ⁽ᶜ⁾` route therefore needs ~700 lines re-derived.
- **Pivot:** realize the *same* ε₀ order as a **recursive multiset (path) order `HLt`**
  built on mathlib's existing `Relation.CutExpand` (`Mathlib/Logic/Hydra.lean`). This is
  cleaner than re-deriving nadd AND answers that file's literal `TODO: formalize …
  Kirby–Paris hydras`. The ordinal `o` is the *rank* of `HLt`; the docstrings say so, so
  the "ε₀ ordinal-descent" framing is preserved honestly without constructing ordinals.
- **Datatype:** nested-through-`Multiset` is kernel-rejected (positivity on a quotient).
  Used `Hydra := node : List Hydra → Hydra` (spec's documented fallback). Game is on
  *unordered* trees → children compared up to `List.Perm` everywhere; no `DecidableEq`
  needed (deliberately avoided — `deriving DecidableEq` doesn't fire on nested inductives
  anyway). Plain `induction` also doesn't fire → custom `@[induction_eliminator]
  Hydra.ind'`.
- **The one real subtlety** (worth understanding if you ever build a recursive path
  order again): irreflexivity and well-foundedness are normally entangled — the obvious
  inductive `HLt` admits a *phantom self-loop* at every node iff `∃ a, HLt a a`, so
  irreflexivity isn't structurally inductive and the leaf base case can't be proven. Fix:
  **pin the removed child into the relation as `a ∈ s`.** Then a self-loop forces
  `HLt a a` for an actual *child* `a`, so irreflexivity IS a structural induction →
  `Std.Irrefl HLt` → unblocks mathlib's `acc_of_singleton` / `Acc.cutExpand` for
  well-foundedness. All chop moves genuinely have `a ∈ s` (the removed head/node is a
  real child), so the constraint costs nothing.
- Anchors are **explicit constructor-witness `example`s**, not `decide`/`native_decide`
  (the relation is `Prop` over `Perm`, deliberately non-computable). These are
  axiom-clean and strictly stronger than `decide` (they exhibit real derivations).
