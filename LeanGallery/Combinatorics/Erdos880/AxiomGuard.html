    Axiom guard (regression test) #

    #print axioms only emits an info message — a sorry/axiom regression would still build. The #guard_msgs wrappers below pin the exact axiom list of every load-bearing theorem, so if anyone ever reintroduces sorryAx (or any new axiom) the build fails here. This is the permanent, build-enforced statement of "the whole resolution is kernel-pure".

    Expected axioms for every headline: [propext, Classical.choice, Quot.sound] — the three mathlib foundations, nothing else (no sorryAx, no native_decide/ofReduceBool, no cited axiom).
