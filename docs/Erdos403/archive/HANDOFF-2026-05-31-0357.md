  "A power-of-two factorial sum with a tied bottom pair exceeds its top index by at most 2."
  Explicit `B = 2`. Everything else is proven *from* it (`tied_carry_ceiling = ⟨2, tied_sharp_ceiling⟩`
  → finiteness; unique-min `m≤3` + kernel + size sandwich → sharp `m≤7`).

  `erdos_403_sharp` were *two separate* sorries. I unified them — see commit `c77caab`. The sharp
  endgame needs **no** factorial-base layer / `decide` (the old plan assumed it did): unique-min is
  unconditional (`sharp_of_unique_min`, `m≤3`), tied is kernel + `four_two_pow_lt_factorial`
  (`2^{M+2}<M!` for `M≥6` ⟹ `M≤5` ⟹ `m≤7`).
- **`B=2` is sharp AND the `factSum=2^m` hypothesis is load-bearing** (don't try to drop it): the
  *general* gap `v₂(factSum S) − max'S` is **unbounded** — family `{2ᵗ−2, 2ᵗ−1, 2ᵗ+1}` gives gap
  `2t−2` (`{6,7,9}`→4, `{14,15,17}`→6). So no constant `B` works without the power-of-2 constraint.
  The carry jumps are governed by *odd-part ratios* of factorials (`oddpart(9!)/oddpart(6!·7!) =
  2835/45 = 63 = 2⁶−1`). This is *why* it's Lin's hard estimate — don't chase a crude general bound.
- **Bottom is pinned:** `min'_le_two` (proven) + tied ⟹ `min'S ∈ {0,2}`. The `a₀=0` case reduces to
  `a₀=2` via the `0!+1!=2!` twin; the `{0,1,2}⊆S` sub-case dies by a mod-8 parity (`m≥3` ⟹
  `4+6·[3∈S] ≢ 0 mod 8`). So the live core is **`a₀=2`, `{2,3}⊆S`, bottom `=8`.**
- **The cascade proof strategy (this is Lin's argument — the path to prove the kernel).** For `a₀=2`:
  `2^m = 8 + ∑_{a≥4,a∈S}a!`; divide by 8 → `2^{m-3} = 1 + ∑_{a≥4} a!/8`, where `a!/8` is *odd* iff
  `a∈{4,5}`. Parity forces **exactly one of `{4,5}` in S**, and the equation *recurses one pair up*
  with the exponent shifted (take 4 → advance 2 levels; take 5 → bigger jump). Terminates when S is
  exhausted at `m ≤ max'S + 2`. **Formalizing this finite-depth descent is the kernel.** It needs a
  *generalized* inductive predicate (the `2^t = carry + ∑ a!/divisor` form changes shape each level,
  so you cannot recurse `tied_sharp_ceiling` itself) — that's why I didn't start it at session-end.
  Likely strong induction on a "tail set." Multi-session.
