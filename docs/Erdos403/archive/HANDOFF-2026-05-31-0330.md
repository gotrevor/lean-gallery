## 🧠 Context to carry forward
- **The reframing is everything.** `factSum S = 2^m` is impossible once *both* `2^m` and `2^m−1`
  carry a factorial digit `≥ 2` (interface lemma `not_factSum_of_digits`). So both headline theorems
  reduce to a digit question on `2^m`. This **supersedes** the old `tied_carry_ceiling` cascade —
  once Phase C lands, prove `erdos_403_sharp` via FNS, then reroute `erdos_403_finite` through sharp
  and *retire* `tied_carry_ceiling` (Phase D9). Don't sink more effort into the tied-pair cascade.
- **Why FNS works**: a number is a sum of distinct factorials (idx ≥ 1) iff all its factorial digits
  are ≤ 1; `0!=1!` adds one optional unit → the "`n` or `n−1`" branch. Proven in `FactBase`.
- **Parity split (verified against brute enumeration, m=1..15, trust it):**
  - **Even `m ≥ 4`: DONE.** `2^m ≡ 16 (mod 24)` ⟹ `d_3 = 2` on *both* `2^m` and `2^m−1`; `3!=6`
    has no degeneracy so the `0!` carry can't fix it. → `factSum_ne_of_even`.
  - **Odd `m`: small digits are useless** — `d_2, d_3 ≤ 1` on both numbers. (My session-2 PLAN note
    claiming odd `m` loses a branch to `d_3` was WRONG; I corrected it via computation. Don't trust
    hand-computed digit claims — `python3` brute-checks are cheap, use them.)
- **The real nut (Phase C, ~50%)**: odd `m ≥ 9` needs a *higher* digit ≥ 2 in both `2^m` and `2^m−1`.
  Two regimes by leading digit `d_M = ⌊2^m/M!⌋` (M = top factorial index ≤ 2^m):
  (a) `2^m ≥ 2·M!` → leading digit ≥ 2, **provable** with `two_le_factDigit_top` (already added);
  (b) `2^m ∈ [M!, 2M!)` → need a *middle* digit ≥ 2 (e.g. m=13: lead=1 but `d_6=4`). This is the
  irreducible Lin core — but now stripped of the `0!` wrinkle and the tied/untied split.
- **Abandoned**: bounding `v₂(K)` (circular, `v₂(K)≈m`); any cheap crude bound on the cascade
  (`v₂(factSum)` is unbounded over general tied pairs; only odd-part-=1 tames it).
