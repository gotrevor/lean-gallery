## 🧠 Context to carry forward
**The reframing that unlocked everything:** `cascade_two` concludes `m ≤ M+2` (M = `max' S`). For
`M ≥ 6` the sandwich *already* forces `m ≥ M+3` (because `2^{M+2} < M!`), so `m ≤ M+2` can only hold
**vacuously** — i.e. the `M≥6` branch is really "derive `False`". Don't try to *bound* `m` there;
derive a contradiction.

**The mechanism that won most branches — valuation gaps.** Write `factSum = head_n + tail_n` at split
`n`. Since `head+tail = 2^m` (both positive, `m` large), `v₂(head_n) = v₂(tail_n)` is *forced* at
every `n` (else the min-valuation survives and `v₂(sum) < m`). Call it the carry `c_n`. The achievable
`v₂` of a factorial tail `∑_{a≥n} a!` has **gaps**: it's `v₂(n!)` (exactly one of the pair `{n,n+1}`)
or `≥ v₂((n+2)!)` (both/neither) — nothing strictly between. If the forced carry `c_n` lands in a gap
`(v₂(n!), v₂((n+2)!))`, contradiction. That's a mod-`2^k` check (terms above a threshold vanish mod
`2^k`).

**What's proven (each a closed branch of `tied_sharp_ceiling` → `cascade_two`):**
- `max' S ≤ 2`: `factSum ≤ 0!+1!+2! = 4 ⟹ m ≤ 2`.
- `a₀ = 0` (both `2∈S` via mod-8 kill, and `2∉S` via the `0!+1!=2!` twin surgery) — prior session.
- `a₀ = 2, M ≤ 5`: pure sandwich (`M! < 2^{M+2}` for `M≤5`, by `decide`).
- `a₀ = 2, M ≥ 6`: sandwich ⟹ `m ≥ M+3 ≥ 10`. Parity (mod 16) ⟹ **exactly one of `{4,5} ∈ S`**.
- `4 ∈ S` branch: forced carry `v₂ = 5` lands in gap `(v₂(6!),v₂(8!)) = (4,7)` ⟹ `factSum ≡ {16,32,48}
  ≢ 0 (mod 64)`, contradiction. **Closed** (4 `{6,7}`-membership sub-cases, each via `not_pow2_dvd_factSum`).
