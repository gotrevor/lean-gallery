## 🧠 Context to carry forward
- **The whole of Theorem 3.1 is the cone bookkeeping.** Def 2.2/2.3 split parameter space into 6 cones
  `𝒟₁..₆` (by `(m,l)`), each with sign `±` on `k` ⇒ **12 sub-subcones**. I built a **cone-agnostic
  master** `st06_thm31_closed_core` that takes the even→odd inequality core as a hypothesis, so each of
  the 12 reduces to one `*_core` interval lemma (~30 lines). The master's odd→even step is exact and
  shared; it needs only `t+mg ≠ 0`, so it serves both `Ω₁` (`m≥1`, `P=t+mg>0`) and `Ω₂` (`m≤−2`, `P<0`)
  unchanged. **This is the key structural insight — don't re-derive per cone.**
- **The reduction is universal**: every cone's even→odd step collapses to `0 ≤ l/(g−1)+a(ε−f) < 1`
  with `a = klg/((g−1)(t+mg))`, `f∈[0,1)` the fractional part. Only the ε-interval and the sign of
  `a`/`P` differ. Two templates: `d5m_core` (a>0, P<0) and `d5p_core` (a<0, P<0) for Ω₂; the Ω₁ ones
  (`d2m`/`d2p` etc.) for P>0. Sign of `a = sign(kl)`.
- **VERIFY-DON'T-TRUST paid off twice.** (1) `notes/ST06-PLAN.md`'s transcription of the Thm 3.1
  ε-interval had a spurious `+1` on the *upper* endpoint — the correct condition is `1+γᵢ^s ≤ ε < δᵢ^s`
  (lower keeps +1, upper drops it), verified 0-fails over ~250k points across all 12 cones. See
  `notes/ST06-THM31-ERRATUM.md` (has the `γ,δ` line-table). All 12 formalized cores use the corrected
  intervals. (2) Thm 3.3's "dₙ" is ambiguous in the plan — numerically it's indexed with **`d₁` = the
  integer digit** (`dₙ=⌊t·2^{n−1}⌋−2⌊t·2^{n−2}⌋`), same as `Cor13e.lean`. Always run/extend the sandbox
  scripts before formalizing.
- **Lean gotchas hit this lap** (also flushed to memory buffer): `field_simp` can leave a `(…)⁻¹`
  uncleared when its normal form reorders the denominator (`t+m*g` → `g*m+t`) — fix by `rw [show … from
  by ring]` on the denominator *before* `field_simp`. For `P<0` cores, the final division flips:
  `0≤N/D` (D<0) via `rw[←neg_div_neg_eq]; div_nonneg …`; `N/D<1` via `(div_lt_one_of_neg hD).mpr`.
