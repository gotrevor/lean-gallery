## 🧠 Context to carry forward
- **The crux engine that works for these binary recurrences** (the key mental model): each theorem's
  even→odd ("b") and odd→even ("a") steps are floor equalities. Prove them as standalone lemmas
  **abstracted over a free integer `s ≥ 1`** standing for `2ⁿ` — the only fact used is the digit
  relation `2B ≤ C ≤ 2B+1` (`B=⌊t·s/2⌋, C=⌊t·s⌋`). In the algebra **the `s`-terms cancel**, leaving an
  inequality in `y=t·s−C ∈ [0,1)` and `d=C−2B∈{0,1}` only → `nlinarith` + `mul_nonneg` hints close it.
  The master then instantiates `s:=2^j`. (This is why the hard nonlinear b-cruxes were tractable.
  Flushed to memory as `lean-floor-crux-scale-cancels`.)
- **Thm 3.3 vs 3.4 — why 3.4 collapses to ε=½.** Both reduce the b-step to `0 ≤ (1−d) − b(ρ−ε) < 1`.
  In 3.3 the d=0 and d=1 ρ-ranges are **separated** → genuine `t`-universal interval
  `½±(2l+1)/(2(2m+1))`. In 3.4 they **meet at ε=½** → only ε=½ works for ALL `t`; the narrow band
  numerics show for √2 (e.g. `[0.4945,0.5018]`, stable past n=40) is **Diophantine** (√2's digits avoid
  the boundary), the **pair-5 phenomenon**. So I formalized only the honest `t`-universal ε=½ case and
  filed `ON-LINE-REQUEST.md` for the PDF's exact Thm 3.4 interval. (Diagnostic flushed as
  `lean-digit-interval-branches-meet`.) **Don't formalize against Stoll's printed k-dependent interval
  without the paper's per-w argument — that's the pair-5 trap the prior handoff warned about.**
